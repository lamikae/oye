To discover each other, peers need a central hub to coordinate.

Oye is a modest WebRTC hub

Oye is intended for you to take and repurpose for your needs. The AGPL license kindly asks you to give back the changes you feel others might find useful.
The engine of Oye is patched fork of Licode. They provide a solid base and upstream that keeps track of changes in the spec.

Raspbian form-factor

The licode codebase is patched to scale down to run on Raspberry Pi for the benefit of reduced footprint.

A bit wary on all this cloud business?

Q: What strange "cloud technologies" are necessary to run all this? How do I know what they are doing?

A: Nginx and three node.js processes on the server. An install script for Debian GNU/Linux based systems is provided, and what apt-get cannot provide, is built from source code. There are no proprietary components.

Q: Node.js? It must be very slow then.

A: Only the higher level control logic is Javascript. The core modules are written in C++.

Q: What browsers and operating systems are supported?

A: Client browsers may be running on a variety of Linux, Windows and OSX hosts. Not all mixtures may work at all times though. Currently Firefox, Chrome and Opera have integrated features of WebRTC.

Q: Where's the catch?

A: You still need a STUN server...

STUN what?

The Oye system is self-contained; once installed, it does not require resources from the internet if all peers are within the same local network.

To connect with peers on the internet, a STUN server is needed for public ip discovery. Oye does not yet have this feature, and you may consider using the default google server to lookup your ip address.
